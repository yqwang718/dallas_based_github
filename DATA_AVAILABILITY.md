# Data Availability

This replication package includes:

- all code required to run the base Dallas DCM workflow in this repository,
- **synthetic** JSONL data under `data/features/` that preserve the schema of
  the analytic inputs,
- any outputs generated by users from those synthetic inputs.

The package does **not** redistribute the full analytic inputs used in the
paper.

## Raw Source Data (not redistributed)

| Source                                         | Access                                                        | Why not redistributed                  |
|------------------------------------------------|---------------------------------------------------------------|----------------------------------------|
| Dallas Police Incidents                        | Publicly available via the Dallas open-data portal            | Large file; citation in the manuscript |
| Dallas Police Arrests                          | Publicly available via the Dallas open-data portal            | Large file; citation in the manuscript |
| 2010 Decennial Census SF1                      | U.S. Census Bureau                                            | Large; redistribution constraints      |
| 2008–2012 ACS 5-year estimates                 | U.S. Census Bureau                                            | Large; redistribution constraints      |
| 2010 TIGER/Line shapefiles (Dallas, TX)        | U.S. Census Bureau                                            | Large                                  |
| DART transit stops (GTFS via transitland)      | https://www.transit.land/operators                            | Upstream license                       |
| OpenStreetMap points of interest (via OSMnx)   | https://www.openstreetmap.org                                 | ODbL; redistribution restrictions      |

Geocoded home and incident coordinates derived from the raw police records are
also not redistributed, since they can enable re-identification of individuals.

## What the Synthetic Data Are For

The synthetic files under `data/features/` are designed to:

- mirror the JSONL column names and pydantic schemas expected by the base
  pipeline (see `dcm/protocols.py`),
- preserve key cross-file relationships (block group IDs, home/incident block
  linkage),
- allow the full pipeline to run end to end on a small example without any
  data downloads.

They are **not** intended to reproduce the published estimates.

## Replicating Published Estimates

Users with authorized access to the real analytic inputs can drop them into
`data/features/` while keeping the same file names and schemas, then re-run:

```bash
bash scripts/base/run_base.sh
```

No code changes are required. See `REPLICATION_NOTES.md` for the exact
schemas and file relationships.

## Preprocessing Pipeline

If you are starting from raw Dallas PD CSVs, the
`scripts/base/preprocessing/` folder documents and implements the full
pipeline from raw addresses → analytic JSONL. A sample raw-data generator is
included so the pipeline can be tested end to end without real data.
